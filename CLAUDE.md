## Derived files - always regenerate after changes

`research/articles.json`, `research/index.html`, `sitemap.xml`, and `feed.xml` are generated by:

```
python3 tools/site-build/build_indexes.py
```

The pre-commit hook at `.githooks/pre-commit` runs this automatically and stages the output.
Never edit these files by hand.

## Adding a new article

1. Create `research/<slug>.html` using an existing article as the scaffold.
2. Add entry to the manifest section in `tools/site-build/build_indexes.py` (or `research/articles.json` if that's the source).
3. Run `python3 tools/site-build/build_indexes.py` - regenerates index, sitemap, feed.
4. Add prev/next cross-links to adjacent articles.
5. If the article has Sigma rules, add them under `tools/sigma-pack/rules/<platform>/`.

Each article: real sourced content, 1900-2400 words, working detections (Sigma/KQL), MITRE ATT&CK mapping, honest limitations section. Not placeholder.
